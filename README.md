# MPI 并行计算作业与课设

中国地质大学（武汉）并行计算课程作业及课程设计。

## 目录结构

| 目录 | 说明 |
|------|------|
| `MPI_hello/` | MPI 入门：并行归并排序（二分归并） |
| `homework2/` | 同 MPI_hello，独立作业副本 |
| `homework3/` | MPI 矩阵乘法：3×3 矩阵，9 进程各算一个元素 |
| `homework4/` | 基于 MPI_Send/Recv 实现 MPI_Gather（HMPI 风格） |
| `homework5/` | 基于 MPI_Send/Recv 实现 MPI_Allgather（环形算法 + HMPI） |

## 思政思考：华为在高性能/并行计算领域的核心竞争力

华为在 HPC/并行计算领域最大的竞争力在于 **"芯片—框架—编译器"全栈自研的并行计算生态**，与课程内容直接对应：

| 课程概念 | 华为对应能力 |
|---------|-------------|
| MPI 集体通信（Gather、Allgather、Reduce 等） | **MindSpore 框架** 内置的分布式并行策略（数据并行、模型并行、流水线并行），自动在集群中完成梯度 Allreduce、参数 Gather 等操作 |
| 进程间通信、拓扑结构 | **HCCS（华为高速互联）** 实现昇腾芯片间低延迟通信，类似 MPI 中 communicator 的物理层 |
| 并行效率、负载均衡 | **昇腾 Ascend** 的达芬奇架构（DaVinci Core）专为矩阵运算并行设计，3D Cube 单元单次可完成 16×16×16 的矩阵乘加 |
| 编译器优化 | **毕昇编译器（Bisheng）** 针对鲲鹏/昇腾自动向量化、循环并行化，类似 MPI 代码的手动优化被编译器接管 |

相比竞品，华为的独特壁垒是**软硬协同**——从物理芯片到通信库再到训练框架全栈打通，不依赖第三方组件即可完成大规模并行计算。这与课程中 MPI 只做"通信标准"不同，华为向下控制了互联硬件，向上优化了并行策略，形成完整的闭环竞争力。
