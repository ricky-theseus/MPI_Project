\section{结论与展望}

\subsection{工作总结}

本实验以旅行商问题（TSP）的标准测试实例 pcb442 为研究对象，基于 MPI 并行编程框架实现了三种不同架构的分布式进化算法——主从式 dEA、分层式 HdEA 和带环形迁移的 HdEA-MP——在严格可控的实验条件下（相同参数、相同硬件、相同编译器优化级别）各运行 10 次，并通过多重统计指标进行了系统比较。

实验结果明确地揭示了主从式架构在本实验设定下的优势：dEA 以 51606 的均值超过了已知最优解 50778 的约 1.40\%，远优于 HdEA（56596，高 9.7\%）和 HdEA-MP（76036，高 47.4\%）。统计分析进一步证实了三种算法之间的差异具有高度的统计显著性（配对 t 检验 $p < 10^{-8}$）。

进一步分析指出，造成这种性能差异的根本原因在于有效种群规模和迁移策略的差异。在总个体数固定的条件下，dEA 的全局大种群在维持遗传多样性和搜索效率方面优于 HdEA/HdEA-MP 的岛屿分割策略。四层岛屿模型在每岛仅 50 个个体的极端条件下，岛屿内的遗传多样性快速枯竭，而迁移机制（尤其是 HdEA-MP 的环形迁移）由于设计上迁移量过大或迁移频率过高，反而破坏了对不同搜索区域的独立探索，导致解的质量恶化。

\subsection{不足与改进方向}

本实验在资源受限的条件下（单台 4 核机器）开展，虽然尽量控制了变量，但仍存在以下局限性：

\begin{itemize}
    \item \textbf{硬件限制：}所有实验仅在 4 进程的配置下运行，未能测试算法扩展到更多进程（如 8、16、32）时的性能变化。特别是岛屿模型算法在更多进程时，每个岛屿的规模将进一步减小，性能恶化可能更加严重。
    \item \textbf{迁移参数未经调优：}HdEA-MP 的迁移频率（20 代）和迁移数量（3 个）采用了经验值，未进行系统性的参数扫描。更低的迁移频率或更小的迁移量可能带来更好的结果。
    \item \textbf{缺少与精确解的对比：}虽然已知最优解为 50778，但本实验未使用精确求解器（如 Concorde）在相同机器上验证运行时间，缺少"精度 vs. 时间"的权衡分析。
\end{itemize}

\subsection{未来工作}

在本次实验的基础上，以下方向值得进一步探索：

\begin{enumerate}
    \item \textbf{自适应迁移策略：}设计一种动态的迁移触发机制——例如，仅在岛屿内的种群多样性下降到预设阈值时才触发迁移，而非每隔固定代数强制迁移。这将使迁移作为一种"救援"机制而非常规操作，有可能在保持多样性和稳定进化之间取得更好的平衡。
    
    \item \textbf{异构迁移拓扑：}将 HdEA-MP 的固定环形拓扑扩展为动态拓扑或随机拓扑（如随机选择迁移伙伴），降低相邻岛屿之间解的同质化风险。
    
    \item \textbf{多实例验证：}在多个 TSPLIB 实例（如 pr1002, rat783, d1291）上重复实验，验证三种算法的性能表现是否具有可推广性，而非 pcb442 的特例。
    
    \item \textbf{混合并行：}将 OpenMP 共享内存并行与 MPI 消息传递结合，在节点内使用 OpenMP 并行化适应度评估，在节点间使用 MPI 进行岛屿迁移，以适应异构计算环境和大规模并行。
\end{enumerate}

\subsection{经验教训}

从本次实验中，我们获得了若干关于并行进化算法设计的实践经验：

\begin{itemize}
    \item 并行架构的选择不应脱离问题规模独立考虑。对于中等规模的 TSP 问题（442 城市），总个体数 200 时主从式架构远优于岛屿模型——"全局即最优"在有限资源下是一个强有力的设计原则。
    \item 迁移不是越多越好。"移民"对接收方种群的影响是双向的：既能引入新鲜基因，也可能破坏本地进化积累的有益模式。迁移策略需要精细调整迁移量、迁移频率和替换策略，三者之间的相互作用决定最终效果。
    \item MPI 通信模型的选择需要充分考虑算法的并发特性。使用 MPI\_Sendrecv 替代交替的 MPI\_Send/MPI\_Recv 可以有效避免死锁，体现了并发编程中"一个操作完成一项原子性通信"的设计智慧。
\end{itemize}

综上所述，本实验系统比较了三种主流并行 EA 架构在 TSP 上的表现，为并行进化算法的架构选择和参数设计提供了有价值的实证参考。
