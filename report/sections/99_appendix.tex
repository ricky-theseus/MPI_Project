\subsection{用 MPI\_Send / MPI\_Recv 实现 MPI\_Gather}

\noindent 不使用 MPI\_Gather，仅用 MPI\_Send / MPI\_Recv 实现自定义 \texttt{my\_MPI\_Gather}，并与标准 MPI\_Gather 对比验证。

设计上遵循 HMPI 规范——root 进程通过 \texttt{memcpy} 将自己数据直接拷贝到接收缓冲区的对应位置，其余进程通过 \texttt{MPI\_Send} 将数据发送至 root，root 按 rank 顺序通过 \texttt{MPI\_Recv} 依次接收。此外，程序通过分布式打印演示了 Gather 的核心语义：将字符串 ``姓名: 任天翌~~学号: 20231003512'' 切分成 4 段分发给 4 个进程，各进程调用 \texttt{my\_MPI\_Gather} 将各自持有的段汇聚至 rank 0，rank 0 拼接后输出完整字符串，同时保留整数版本的验证测试。

\subsection{用 MPI\_Send / MPI\_Recv 实现 MPI\_Allgather}

\noindent 不使用 MPI\_Allgather，仅用 MPI\_Send / MPI\_Recv 实现自定义 \texttt{my\_MPI\_Allgather}，并与标准 MPI\_Allgather 对比验证。

采用环形轮转算法（Ring Allgather）：每个进程先将自身数据块写入接收缓冲区的对应位置（HMPI），随后进行 $size-1$ 步环形传递——每步当前进程向右邻居发送一个数据块，同时从左邻居接收一个数据块，逐步完成全交换。同样通过分布式打印演示了 Allgather 的核心语义：与 Gather 不同，所有进程最终都获得完整字符串，体现了 Allgather = Gather + Bcast 的聚合广播特性。

% ===== 思政思考 =====
\subsection{思政思考：华为在高性能/并行计算领域的核心竞争力}

\begin{enumerate}
\item 华为构建了"芯片—框架—编译器"全栈自研的并行计算生态。鲲鹏处理器提供 ARM 架构通用算力，昇腾 AI 处理器的达芬奇架构（DaVinci Core）内置 3D Cube 矩阵计算单元，专为大规模并行矩阵运算设计，单次可完成 $16\times16\times16$ 的乘加操作。

\item 在并行计算框架层面，MindSpore 框架内置了数据并行、模型并行、流水线并行等分布式策略，自动在集群中完成梯度 Allreduce、参数 Gather 等集合通信操作，开发者无需手动编写 MPI 通信代码即可获得高效的分布式训练能力。

\item 在通信层，华为自研的 HCCS（华为高速互联）实现了昇腾芯片之间的低延迟高带宽通信，为大规模并行计算提供了高效的物理互联基础。

\item 在编译优化层，毕昇编译器（Bisheng）针对鲲鹏和昇腾硬件做了深度自动向量化与循环并行化优化，将传统 MPI 编程中需手动调优的环节交由编译器自动完成。

\item 华为的独特壁垒在于软硬协同——从芯片到并行框架全部自主可控，不依赖第三方组件即可完成大规模并行计算。相比 MPI 只定义通信标准的开放模式，华为实现了从底层硬件到上层框架的垂直整合，在当前国际形势下具有重要的战略意义。
\end{enumerate}
