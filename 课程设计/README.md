# 课程设计：TSP 并行计算解决方案

## 目录结构

| 文件 | 说明 |
|---|---|
| `TSP0.C` | 串行 TSP 求解器（群体搜索 + 反转变异） |
| `pcb442.tsp` | TSPLIB 标准测试集，442 个 PCB 钻孔城市坐标 |
| `hierarchical.pdf` | 参考文献（层次化并行 TSP 求解） |
| `TSP0.exe` | 已编译的串行可执行文件 |

## 问题描述

旅行商问题（TSP）：给定 n 个城市及两两间的距离，求经过每个城市恰好一次的最短回路。

测试数据 `pcb442.tsp` 来自 TSPLIB，包含 442 个城市，坐标为一组 PCB 钻孔位置，城市间距离取四舍五入后的欧氏距离（整数）。

## 算法说明

`TSP0.C` 实现了一种基于群体搜索的启发式算法：

- 维护 `xColony` 条随机路径（排列），每条路径代表一个候选解
- 每次迭代对每条路径执行**反转变异**：翻转路径中两座城市间的一段，使路径变化可控
- 变异参考来自：以概率 `probab1` 随机选城市，否则从另一条路径的对应位置继承
- 经过 `xColony` 次迭代后执行选择（父子竞争替换）
- 终止条件：达到 `maxGen` 代

### 运行结果

串行版 60 秒截停的输出：

```
2:720740 → 2000:340177
```

完整收敛需要更长时间。最佳路径距离依赖于参数 `N_COLONY`、`probab1` 等。

## MPI 并行化方向

将 `xColony` 条路径分配到多个 MPI 进程，每轮迭代：

1. **主进程分发**：每个子进程分得一批路径
2. **并行评估**：各进程独立对分配的路径执行变异 + 距离计算
3. **归并结果**：子进程将改进后的路径和适应度传回主进程，主进程执行全局选择

## 编译与运行

### 串行版

```bash
cl /TC /O2 TSP0.C
TSP0.exe
```

注意输入文件路径硬编码为 `d:\pcb442.tsp`，输出文件为 `e:\tsp0.txt`，如路径不适配需修改源码。
